# Документация по меткам и тегам

Этот документ объясняет, как направлять ИИ-анализ специализированным ролям с помощью определенных тегов в сообщениях коммитов или меток в Pull Request. Разделяя изменения по категориям, скрипт фокусируется на конкретных аспектах кода для создания максимально релевантных задач (Issue). Чтобы эффективно отслеживать и управлять сгенерированными задачами, вам нужно работать с таблицами в GitHub Projects.

## Поддерживаемые роли и метки

### 1. Строгий аудитор безопасности (Strict Security Auditor)
* **Триггеры (Метки/Теги):** `security`, `sec`, `audit`
* **Фокус:** Глубокий аудит безопасности на основе OWASP Top 10.
* **Результат:** Сканирует код на наличие инъекций, XSS, захардкоженных секретов и небезопасной обработки данных. Создает критический отчет об уязвимости с указанием конкретных строк кода и инструкций по их исправлению.

### 2. Строгий Code Reviewer (Strict Code Reviewer)
* **Триггеры (Метки/Теги):** `review`, `refactor`, `code-review`
* **Фокус:** Качество кода, принципы SOLID, DRY и общая архитектура.
* **Результат:** Выявляет "запахи кода" (code smells), плохие соглашения об именовании и избыточную логику. Генерирует Issue с конкретными предложениями по рефакторингу и примерами более чистого кода.

### 3. QA Инженер (QA Engineer)
* **Триггеры (Метки/Теги):** `qa`, `test`, `testing`
* **Фокус:** Граничные случаи (edge cases), точки отказа и тестовое покрытие.
* **Результат:** Генерирует готовые блоки кода для unit-тестов и маркированный чек-лист для ручного тестирования, чтобы убедиться, что новые изменения не ломают существующий функционал.

### 4. Эксперт по производительности (Performance Expert)
* **Триггеры (Метки/Теги):** `perf`, `optimize`, `performance`
* **Фокус:** Узкие места производительности, временная/пространственная сложность и управление ресурсами.
* **Результат:** Выявляет медленные циклы, избыточные запросы к БД или высокое потребление памяти. Предлагает алгоритмические оптимизации для повышения скорости выполнения.
